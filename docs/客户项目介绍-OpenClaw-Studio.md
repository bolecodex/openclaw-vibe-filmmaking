# OpenClaw Studio 项目介绍（客户版）

> **一句话**：把小说或剧本，一步步变成**带画面、有配音、可预览的 AI 视频短片**，并在统一工作台里管理全流程与资产。

---

## 一、解决什么问题？

传统漫剧 / 短剧 / 视觉化叙事制作，往往要在多个工具之间来回切换：写脚本、画分镜、约配音、做动效、最后合成。  

**OpenClaw Studio** 把这条长链路收拢到一个产品里：

- **按步骤推进**：从文本到分镜、从分镜到图、从图到声、从声到动态视频，再到成片合成。  
- **资产可复用**：角色、场景、道具等形成「项目资产库」，保证前后画风与设定一致。  
- **人机协作**：既可以用界面上的「流水线」点选执行，也可以用右侧对话让智能助手代为触发步骤。  
- **过程可追溯**：执行日志帮助了解每一步是否完成、哪里需要重跑。

适合：**IP 方、内容团队、MCN、有小说/剧本需视觉化**的客户，作为内部产能工具或与客户演示的制片工作台。

### OpenClaw（龙虾）在方案里做什么？

- **OpenClaw Studio**：您看到的**工作台**——管理项目、分镜、图音视频预览、流水线按钮与执行日志。  
- **OpenClaw Gateway（业内常口语化为「龙虾」运行时）**：背后的**智能体引擎**——按预置 **Skill** 把「切场景、出图、配音、做视频、合成」等步骤变成**可重复调用、可观测、可重跑**的任务链，并安全地访问各家多模态模型 API。  
- **价值一句话**：创意与审片留在 Studio；**累活、编排与对模型的连续调用**交给 Gateway，避免成果散落在无审计的聊天窗口里。

### 什么是 Vibe Filmmaking（氛围驱动制片）？

先定**整体气质、节奏与情绪（vibe）**，再迭代具体镜头与音画，而不是一开始就钉死每一帧细节。  
本方案通过**同一工作区内的单步重跑、资产库复用、对话式局部调整**，支撑「试 vibe → 收敛风格 → 延展镜头 → 合成交付」，**缩短从灵感到可播成片的路径**。

## 二、您最终能得到什么？（交付物视角）

| 类型 | 说明 |
|------|------|
| **结构化剧本与场景** | 场景切分、场景说明文档，便于分工与审阅。 |
| **角色 / 场景 / 道具设定** | 统一描述与参考，减少「前后人设不一致」。 |
| **分镜级脚本** | 每个镜头的文案、时长、类型等，可对接拍摄或全 AI 流程。 |
| **分镜静帧图** | 基于大模型图像能力生成的镜头画面（可按项目配置模型）。 |
| **对白与旁白音频** | 按角色音色生成的配音文件。 |
| **镜头级 AI 视频** | 由参考图或文案驱动的短视频片段。 |
| **合成成片** | 将图、声、视频片段按时间线合成，输出可播放的成片文件。 |

具体质量与成本取决于：**所选模型、分辨率、时长、是否反复精修**。产品提供的是**流程与编排能力**，而非单一「一键成片」黑盒。

## 三、创作流程怎么走？（八步，从客户能懂的说法）

整体是**流水线**：上一步的输出，自然成为下一步的输入。

1. **准备文本** — 上传或编辑小说 / 剧本（`.txt` 等）。  
2. **剧本 → 场景** — 自动切场景、整理场景文档。  
3. **提取角色** — 从剧本中归纳角色，形成可引用的角色卡。  
4. **场景 → 分镜** — 在场景内拆成一个个镜头，并生成配图用的提示词等。  
5. **分镜出图** — 为每个镜头生成画面。  
6. **分镜配音** — 为台词与旁白生成语音。  
7. **AI 视频** — 在静帧或文案基础上生成动态镜头。  
8. **合成输出** — 时间线合成，导出成片（如 MP4）。

客户只需理解：**从左到右、从文本到成片**；中间任一步都可以**单独重跑**（例如只重出某一镜的图或视频）。

## 四、系统架构（清晰、通俗版）

用**四层**说明即可，对应「您看到什么 → 背后谁在干活」：

```
┌─────────────────────────────────────────────────────────────┐
│  ① 工作台（浏览器）                                           │
│     项目文件、分镜看板、图 / 音 / 视频预览、对话与任务日志      │
└───────────────────────────────┬─────────────────────────────┘
                                │ 网页与服务器通信
┌───────────────────────────────▼─────────────────────────────┐
│  ② 应用服务（后端 API）                                       │
│     读写项目文件、触发流水线、对接智能体网关、合成与渲染接口     │
└───────────────────────────────┬─────────────────────────────┘
                                │ 任务下发与状态回传
┌───────────────────────────────▼─────────────────────────────┐
│  ③ 智能体与技能（Agent + Skills）                            │
│     每一步对应一个「技能」：切场景、出图、TTS、视频生成等        │
└───────────────────────────────┬─────────────────────────────┘
                                │ 调用模型与工具
┌───────────────────────────────▼─────────────────────────────┐
│  ④ 模型与存储                                                │
│     图像 / 语音 / 视频等大模型能力；项目目录下的 YAML、媒体文件   │
└─────────────────────────────────────────────────────────────┘
```

**对客户说明时的要点**：

- **工作台**：您的「制片桌面」，所有资产和进度都在这里。  
- **应用服务**：保证数据落在您的项目空间里，而不是散落在聊天窗口里。  
- **技能**：把复杂操作封装成标准步骤，便于批量、可重复执行。  
- **模型层**：可按商务与合规要求替换或对接不同供应商（需技术侧配置）。

更偏技术的分层图与接口说明，可参考仓库内 **`architecture.html`**（适合研发或集成商阅读）。

## 五、界面与使用方式（客户演示时可讲）

- **多 Tab 视图**：概览、流水线总控、风格、角色、场景、道具、分镜图、音频、视频、剪辑等，按工种或按工序切换。  
- **流水线面板**：适合「按步骤验收」——看清当前在第几步、哪些镜头已完成。  
- **对话区**：用自然语言下达指令（例如针对某镜头重新生成），适合制片或编导习惯口头协调的场景。  
- **执行日志**：便于排查「这一步是否跑完、是否报错」，减少黑盒感。

## 六、部署与前提（商务 / IT 需知）

实际落地时，通常需要满足：

| 项目 | 说明 |
|------|------|
| **运行环境** | 现代浏览器；服务器侧常见为 Node.js 等（以技术交付清单为准）。 |
| **智能体网关** | 需可访问的 OpenClaw Gateway（或等价能力），用于执行 Skills。 |
| **模型与账号** | 图像、TTS、视频等 API 需按贵司合规流程申请与配置。 |
| **工作空间** | 每个剧目/项目对应一块目录，存放剧本、YAML、生成的图音视频。 |

具体端口、环境变量、依赖列表以 **`README.md`** 与 **`openclaw-studio`** 内说明为准；客户版文档不绑定具体版本号，避免与升级不同步。
