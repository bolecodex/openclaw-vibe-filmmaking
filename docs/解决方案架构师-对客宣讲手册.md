# OpenClaw Studio 用一块工作台把小说或剧本做成带分镜、配音、视频和成片，并由 OpenClaw Gateway（业内称「龙虾」）在后台把多模态 AI 工序编排成可重复执行、可留痕的标准流程。

**适用对象**：制片人、内容负责人、项目决策人；若需技术评审，文后附有分层架构示意图。  
**配图说明**：下文插图为矢量图（SVG），与本文同目录下的 `solution-handbook-diagrams` 文件夹一并提供；转发时建议整夹打包，以免图片丢失。

## 一、您最关心的几件事

| 您可能遇到的麻烦 | 方案能帮您做到 |
|------------------|----------------|
| 脚本、分镜、出图、配音、剪辑散落在不同工具里 | **一个界面**里跑通主线，少来回导出导入 |
| 前后镜头人物、场景「不像一部戏」 | **角色、场景、道具**做成可引用的资产，全局对齐 |

| 改一句词就要从头做 | **按步骤、按镜头重跑**，不用从零开始 |
| 内容存在哪、谁负责审核 | 数据在**您可控的环境**里落地；**对外发布仍由贵司按流程审核** |

## 二、「龙虾」是什么？怎么在您项目里干活？

**口语里的「龙虾」**指的是 **OpenClaw** 这套智能编排能力；落到产品上主要是两件事分工：

| | **OpenClaw Studio**（您操作的网站） | **OpenClaw Gateway**（后台「龙虾」） |
|---|--------------------------------------|----------------------------------------|
| **谁在用** | 编导、制片、美术、运营 | 自动执行，不单独「登录」 |
| **干什么** | 选项目、点流水线、看分镜和音视频、看任务进度、用对话说「把这一镜重配一下」 | 接到任务后，按设定好的**技能步骤**，去调用画图、配音、视频等 AI 能力，并把结果**写回您的项目文件夹** |

可以把它理解成：**您在台前定调、验收；龙虾在台后把一连串 AI 调用排好队、跑完、存好**——而不是每个人各自打开聊天框问一句算一句，版本对不上、也没法审计。

**一次任务大致怎么走（五步）**

1. 您在 Studio 里点某一步（例如「分镜出图」），或用对话说明要改哪一镜。  
2. 系统把当前项目里的剧本、分镜、角色设定等**整理好上下文**，避免 AI「只见一句话、不知全剧」。  
3. **Gateway（龙虾）**按步骤选用对应的**技能**，决定先做什么、后做什么、结果存到哪里。  
4. 去调用您已经接好的**画图 / 配音 / 视频 / 大语言模型**等服务（由贵司账号与合同管理）。  
5. 生成的图片、声音、视频和更新后的数据**回到项目目录**，页面上马上能预览；**同一镜可以反复生成直到满意**。

## 三、Vibe Filmmaking（氛围驱动制片）是什么意思？

很多团队希望 **先统一「这部戏的气质」**——画面像不像、节奏对不对、情绪到不到位——再往下抠每一句词、每一个镜头。**Vibe Filmmaking** 说的就是这种工作方式：先对齐 **vibe（氛围）**，再迭代镜头与音画，减少「一开始就钉死细节、后面整段推翻」的浪费。

本方案用 **同一套资产（角色/场景/风格）**、**可重跑的单步和单镜**，以及 **对话里随时改一小处**，把「试气质 → 试镜头 → 定稿 → 延展 → 合成」尽量留在**同一个工作台**里完成。

![先氛围、再试镜、再收敛交付](solution-handbook-diagrams/vibe-filmmaking-loop.svg)

**和传统写法的区别（一眼看懂）**

| | 传统：脚本先完全定稿再做 | Vibe 取向 | 在本产品里 |
|---|--------------------------|-----------|------------|
| 起点 | 逐字逐镜锁死 | 先定整体气质和节奏 | 先把**风格、角色、场景**立住，再铺分镜 |
| 改一版 | 经常牵一大片 | 小步改气质或单镜 | **只重跑需要的那一步、那一镜** |
| 协作 | 靠口头和文件传来传去 | 中间结果要看得见 | **预览 + 日志**，方便对齐 |

试气质时往往会**多生成几版**；龙虾的作用是让每一版都留在**同一个项目、同一条可追溯链路**里，定调之后后续镜头能**自动沿用同一套人物和场景设定**，方便从「试拍」变成「批量出片」。

## 四、方案优势（您为什么要看这一套，而不是只用一个聊天机器人）

1. **工序是标准的，不是随机的** — 切场景、出图、配音、视频、合成等被做成**可重复跑的步骤**，适合团队协作和外包对接。  
2. **成果在项目上，不在聊天记录里** — 文件落在**项目工作区**，方便管理版本、备份和权限。  
3. **先试整体氛围，再抠细节** — 与 Vibe 制片方式一致，减少无效返工。  
4. **哪一步卡了、要重跑哪一镜，都清楚** — 制片能管进度，不必猜 AI「有没有在跑」。  
5. **不锁死某一家模型** — 具体用哪家画图、配音、视频 API，可按贵司商务与合规更换（实施时配置即可）。

**和常见做法比一眼表**

| | 多个工具拼着用 | 只用通用 AI 对话 | OpenClaw Studio + 龙虾 |
|---|----------------|------------------|-------------------------|
| 工作界面 | 散 | 难管理 | **一个工作台** |
| 流程 | 靠人肉 SOP | 每次结果不稳定 | **流水线 + 固定技能步骤** |
| 资料沉淀 | 容易丢、难对齐 | 容易散在对话里 | **项目里统一存放** |
| 出片 | 常要再换剪辑工具 | 很难端到端 | **支持在流程内合成成片**（以实际交付为准） |

## 五、从 IP 到成片：业务上怎么走、谁关心什么

## 六、产品上长什么样：一块屏幕里完成主线

左侧是**项目和文件**；中间是**功能分区**（如概览、流水线、风格、角色、场景、道具、分镜图、音频、视频、剪辑等，**以实际交付界面为准**）；右侧或周边是**流水线控制台、对话、执行日志**。意思是：**不用为了不同阶段换一整套软件**。

![产品工作台示意](solution-handbook-diagrams/product-workbench.svg)

**从立项到交片，您可以按这个顺序理解使用**

1. 建项目（一部剧、一季内容等）。  
2. 放入或更新小说 / 剧本。  
3. 按界面上的流水线推进：分场、角色、分镜、出图、配音、视频、合成等。  
4. 在各分区里看图、听音、看视频。  
5. 不满意的步骤或某一镜，**单独重跑**或用对话说明要改什么。  
6. 导出成片和素材，走贵司内部的审片与发布流程。

## 七、技术和部署（给需要过一眼 IT / 采购的人）

整体可以想成四层：**您用的网页 → 后面的应用服务 → 龙虾和技能编排 → 外面的模型服务和存放文件的存储**。这样技术同事能评估接口、网络和部署位置。

![技术分层（对客示意）](solution-handbook-diagrams/tech-four-layers.svg)

部署上可以简化成三块：**用户用的浏览器**、**贵司机房或云里的 Studio + 龙虾**、**对外的模型等服务**（密钥和费用由贵司侧管理）。是否专网、是否私有化推理，可在**项目实施时**与贵司安全要求对齐。

![三类区域：终端、核心系统、外部服务](solution-handbook-diagrams/deployment-trust-zones.svg)

## 八、合规与发布（简要）

- **生成内容对外使用**：请仍按贵司制度做**业务与合规审核**，本方案提供的是生产能力，不替代终审。  
- **数据放哪**：项目数据落在贵司约定的存储位置，便于备份和权限管理。  
- **模型服务**：由贵司与各服务商签约与合规；平台负责**对接与编排**，不替贵司承担服务商条款责任。
