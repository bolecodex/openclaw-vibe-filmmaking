OPENCLAW（龙虾）在方案中的角色 · 与 VIBE FILMMAKING 价值

龙虾双核：Studio + Gateway

OpenClaw Studio面向人：项目、分镜、审片、日志，全在浏览器工作台完成。 OpenClaw Gateway面向机：承载 Agent 运行时，按预置 Skill 把「切场景、出图、配音、视频、合成」等步骤变成可重复、可编排的调用链，并经 MCP / HTTP 访问多模态 API。 业内口语中的「龙虾」即指这一整套 OpenClaw 运行时与编排能力；与本方案中的制片工作台成套交付、同域部署，数据与任务不散落在无审计的聊天窗口。

Vibe Filmmaking（氛围驱动制片）

强调先锁定整体气质、节奏与情绪（vibe），再迭代镜头与音画，而非一开始就钉死每一帧细节。 本方案的价值在于：同一工作区内可重跑单步、资产（角色/场景）可复用、对话可触发局部重生成，使「试 vibe → 收敛风格 → 批量延展镜头 → 合成交付」形成闭环； OpenClaw 负责把创意意图稳定翻译成对模型的连续、可追溯调用，降低从灵感到可播成片之间的摩擦与返工成本。

用户终端域

浏览器客户端

  • HTTPS 访问 Studio 前端，操作项目与流水线
  • 默认不在浏览器持久化剧本与成片，降低终端泄露面
  • 企业 SSO 可按项目集成

体验与 Vibe 迭代入口

  • 分镜 / 音视频 Tab 即时预览，快速对齐「气质」
  • 对话与流水线共用同一项目上下文，避免口径分裂

应用与智能体部署域

OpenClaw Studio 应用

  • 前端静态资源 + 后端 API（可同机或水平扩展）
  • 读写项目工作区目录或挂载卷，落盘 YAML / 媒体
  • 与 Gateway 内网互通（地址与鉴权可配置）

OpenClaw Gateway（龙虾运行时）

  • Agent 推理、工具调用、Skill 路由与会话状态
  • 将制片步骤拆解为可观测、可重试的任务单元
  • 建议与 Studio 后端处于同一信任网络
›
外部服务域

多模态模型 API

  • 图像 / 语音 / 视频 / 大语言模型，按合同与合规选型
  • 公网 TLS、专线或私有化推理集群均可对接
  • 密钥、配额与账单归属客户账号体系

对象存储与备份（可选）

  • 工作区可落本地盘、NAS 或 S3 兼容存储
  • 支持异地备份与多环境同步策略
部署策略摘要： 典型路径为「终端 → 客户机房内 Studio + Gateway（龙虾） → 受控出网访问已审批模型端点」； 亦可混合云：计算与数据在客户 VPC，推理走托管 API。端口、高可用与容灾按客户安全基线单独出具详细设计（DD）。
