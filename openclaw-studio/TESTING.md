# 手动测试清单

## 前置条件

1. 启动后端: `cd backend && npm run dev`
2. 启动前端: `npm run dev`
3. 确保至少有一个带 `style.yaml` 的项目目录
4. 打开浏览器访问 `http://localhost:1420`

---

## A. 上下文追踪

- [ ] **A1** 选择项目 → 聊天输入框上方显示 "上下文: {项目名} > 概览"
- [ ] **A2** 切换到角色 tab → 上下文变为 "{项目名} > 角色"
- [ ] **A3** 展开某个角色 → 上下文变为 "{项目名} > 角色 > {角色名}"
- [ ] **A4** 切换到场景 tab → 上下文变为 "{项目名} > 场景"
- [ ] **A5** 展开某个场景 → 上下文变为 "{项目名} > 场景 > {场景名}"
- [ ] **A6** 切换到分镜 tab → 上下文变为 "{项目名} > 分镜"
- [ ] **A7** 展开某个分镜 → 上下文变为 "{项目名} > 分镜 > {分镜ID}"
- [ ] **A8** 发送消息 → 检查后端日志，确认 context 包含 project/view/focus/summary
- [ ] **A9** Agent 回复中体现对当前上下文的理解

---

## B. 图片输入

- [ ] **B1** 截图后在输入框 Ctrl+V → 图片预览条显示缩略图
- [ ] **B2** 点击缩略图的 × → 图片被删除
- [ ] **B3** 拖拽一张图片到输入框 → 预览条显示
- [ ] **B4** 粘贴一张 >2MB 的图片 → 显示大小警告（红色提示 3 秒后消失）
- [ ] **B5** 点击图片按钮选择图片 → 预览条显示
- [ ] **B6** 带图片发送消息 → 用户消息气泡中显示图片缩略图
- [ ] **B7** 发送后图片预览条清空
- [ ] **B8** 检查后端日志，确认 attachments 包含 base64 数据

---

## C. @ 引用

- [ ] **C1** 输入 `@` → 弹出下拉菜单，默认显示全部分类
- [ ] **C2** 输入 `@沈` → 模糊搜索结果过滤
- [ ] **C3** 用上下键选择 → 高亮跟随移动
- [ ] **C4** Enter 确认 → 文本中插入 `@[沈鸢](character:c1)` 格式，下拉关闭
- [ ] **C5** Escape → 下拉关闭，不插入
- [ ] **C6** 点击分类 tab "场景" → 只显示场景列表
- [ ] **C7** Tab 键 → 循环切换分类 tab
- [ ] **C8** 鼠标点击某项 → 选中插入
- [ ] **C9** 发送带 @ 引用的消息 → 消息气泡中显示引用标签（彩色 badge）
- [ ] **C10** 检查后端日志，references 正确传递

---

## D. 端到端场景

- [ ] **D1** 选择项目 → 打开角色页 → 展开"沈鸢" → 输入 `@沈鸢 帮这个角色换发型` + 粘贴参考图 → 发送
- [ ] **D2** 检查后端接收的完整 payload 包含：

  - message
